# 常用指令及操作技巧

● 私有仓库的使用

npm install -g nrm //全局安装nrm
nrm add verdaccio http://x.x.x.x:xxxx //添加私有仓库地址（verdaccio为指定的仓库名）
nrm ls //查看所有镜像源
nrm use verdaccio //切换到私有仓库
npm adduser --registry //注册用户
npm login //用户登录

● nrm 常用命令

nrm //展示nrm可用命令
nrm ls //列出已经配置的所有仓库
nrm test //测试所有仓库的响应时间
nrm add <registry> <url> //新增仓库
nrm use <registry> //切换仓库
nrm del <registry> //删除自定义的镜像源

● npm 包发布更新（基于已有版本发布）

npm version <major | minor | patch> //指定版本
npm publish //发布

● npm 废弃包

npm deprecate <pkg>[@<version>] <comment>

● npm 删除包、删除指定版本

npm unpublish <pkg> --force
npm unpublish <pkg>@<version>

包删除时限

出包管理安全机制，只允许删除 36 小时之内发布的包

● npm 初始化包 package.json、生成默认 package.json（初次发布前执行）

npm init
npm init -y

● npm 等仓库查看及设置镜像源

npm config set registry url //设置镜像源
npm config get registry //查看镜像源

npm 包版本区分

npm 采用语义化版本，共三位，以'.'隔开，从左至右依次代表主版本（major）、次要版本（minor）、补丁版本（patch）。
